# Требования к итоговому проекту

Источник требований: `Требования к проекту.pdf`.

## Выбранная тема

**PlantCare — сервис ухода за домашними и балконными растениями.**

Пользователь добавляет свои растения, получает календарь ухода, отмечает полив
и пересадку, объединяет растения в коллекции и видит рекомендацию по поливу
балконных растений с учетом прогноза погоды. Такой MVP имеет практическую
ценность и не дублирует темы, уже занятые в таблице группы.

Строка для согласования с лектором:

`Лолаев Р. А. | PlantCare: сервис ухода за домашними и балконными растениями | Python + Django REST Framework, React + TypeScript, PostgreSQL, OpenAPI, Open-Meteo API, Docker`

## Стек и границы MVP

- **Frontend:** React, TypeScript, Vite, React Router, Material UI, React Hook Form.
- **Backend:** Python, Django, Django REST Framework, `drf-spectacular` для OpenAPI.
- **База данных:** PostgreSQL; основные сущности: пользователь, вид растения,
  растение пользователя, задача ухода, запись об уходе, коллекция и связь
  растения с коллекцией.
- **Интеграция:** Open-Meteo API для прогноза погоды и подсказки по поливу
  балконных растений.
- **Тестирование:** `pytest` + `pytest-django`, измерение покрытия `coverage`,
  нагрузочные сценарии `Locust`.
- **Запуск:** Docker и Docker Compose.

Реализованные страницы: каталог растений, мои растения, карточка растения,
календарь ухода, профиль/настройки и авторизация.

Реализованные формы: регистрация/вход, добавление растения, создание задачи,
фиксация выполненного ухода, создание коллекции и импорт растений из CSV.

## Итоговая проверка

**Текущий статус:** проект соответствует реализуемым техническим требованиям и
покрывает все пункты дополнительных баллов. Единственное незакрытое действие —
формальное согласование темы/стека с лектором.

- `coverage run -m pytest && coverage report`: 15 passed, backend coverage 92%.

- `docker compose config --quiet`: конфигурация валидна.

- `locust -f locustfile.py --host http://localhost:8000 --headless -u 1 -r 1 -t 5s --only-summary`: 0% ошибок.
- Браузерный сценарий: регистрация, добавление растения, карточка, погодная подсказка, запись ухода, календарь, выполнение задачи, коллекция и переключение темы.
